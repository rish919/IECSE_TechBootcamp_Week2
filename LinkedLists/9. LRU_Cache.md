# LRU Cache (Least Recently Used Cache)

LRU Cache is a classic design problem.

It combines:

- HashMap
- Doubly Linked List
- O(1) operations

This problem tests data structure design and pointer manipulation.


# 1️⃣ Problem Statement

Design a data structure that supports:

- get(key)
- put(key, value)

Constraints:

- Both operations must run in O(1) time.
- When capacity is exceeded, remove the Least Recently Used item.


# 2️⃣ What Does "Least Recently Used" Mean?

Whenever we:

- Access a key (get)
- Insert/update a key (put)

That key becomes "Most Recently Used".

If cache is full:

Remove the key that was used the longest time ago.


# 3️⃣ Why We Cannot Use Only a HashMap

HashMap gives:

✔ O(1) lookup

But it does NOT maintain order of usage.

We need:

- Fast lookup
- Fast removal
- Fast update of order

So we combine:

HashMap + Doubly Linked List


# 4️⃣ Design Overview

HashMap:
- Maps key → Node
- Allows O(1) access

Doubly Linked List:
- Maintains order of usage
- Head → Most Recently Used
- Tail → Least Recently Used


# 5️⃣ Visual Structure

```
Head <-> Node1 <-> Node2 <-> Node3 <-> Tail
```

Where:

Head.next → Most Recently Used  
Tail.prev → Least Recently Used  


# 6️⃣ Node Structure

Each node stores:

- key
- value
- prev pointer
- next pointer

```java
class Node {
    int key;
    int value;
    Node prev;
    Node next;

    Node(int key, int value) {
        this.key = key;
        this.value = value;
    }
}
```


# 7️⃣ Core Operations

We need 4 helper operations:

1. Remove a node from list
2. Insert node right after head
3. Get operation
4. Put operation


# 8️⃣ Full Implementation

```java
import java.util.HashMap;
import java.util.Map;

class LRUCache {

    class Node {
        int key, value;
        Node prev, next;

        Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }

    private int capacity;
    private Map<Integer, Node> map;
    private Node head, tail;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        map = new HashMap<>();

        head = new Node(0, 0);
        tail = new Node(0, 0);

        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) {
            return -1;
        }

        Node node = map.get(key);
        remove(node);
        insert(node);

        return node.value;
    }

    public void put(int key, int value) {

        if (map.containsKey(key)) {
            Node existing = map.get(key);
            remove(existing);
        }

        if (map.size() == capacity) {
            Node lru = tail.prev;
            remove(lru);
            map.remove(lru.key);
        }

        Node newNode = new Node(key, value);
        insert(newNode);
        map.put(key, newNode);
    }

    private void remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void insert(Node node) {
        node.next = head.next;
        head.next.prev = node;

        head.next = node;
        node.prev = head;
    }
}
```


# 9️⃣ How get() Works

1. If key not found → return -1
2. If found:
   - Remove node from current position
   - Insert it at front (most recent)
   - Return value


# 🔟 How put() Works

Case 1: Key exists  
- Remove old node  
- Insert updated node at front  

Case 2: Key does not exist  
- If capacity full:
  - Remove tail.prev (least recently used)
  - Remove it from HashMap
- Insert new node at front  


# 1️⃣1️⃣ Why Doubly Linked List?

We need:

- O(1) removal
- O(1) insertion

Singly Linked List cannot remove a node in O(1) without previous pointer.

That is why Doubly Linked List is required.


# 1️⃣2️⃣ Common Mistakes

❌ Forgetting to remove from HashMap  
❌ Not updating both prev and next pointers  
❌ Removing wrong node (tail instead of tail.prev)  
❌ Not handling capacity edge case  
❌ Forgetting dummy head and tail  


# 1️⃣3️⃣ Edge Cases

✔ Capacity = 1  
✔ Repeated put on same key  
✔ Get on non-existing key  
✔ Put when cache is full  
✔ Multiple consecutive gets  


# 1️⃣4️⃣ Time and Space Complexity

get() → O(1)  
put() → O(1)  
Space → O(capacity)  


# Final Mental Model

Think of it as:

HashMap → Brain (fast lookup)  
Doubly Linked List → Memory order (usage tracking)  

Whenever you use something,
move it to the front.

When memory is full,
remove from the back.

LRU Cache is a perfect example of combining data structures.
