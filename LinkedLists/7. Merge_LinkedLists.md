# Merge Linked Lists

Merging Linked Lists is one of the most important patterns in Linked List problems.

It appears in:

- Merge Two Sorted Lists
- Merge K Sorted Lists
- Sort List (Merge Sort)
- Divide and Conquer problems

If you understand merging clearly, many advanced problems become easy.


# 1️⃣ Merge Two Sorted Linked Lists

## Problem

Given two sorted linked lists, merge them into one sorted list.

Example:

List 1:
1 → 3 → 5  

List 2:
2 → 4 → 6  

Output:
1 → 2 → 3 → 4 → 5 → 6  


# Core Idea

Since both lists are sorted:

1. Compare current nodes
2. Attach smaller node
3. Move pointer forward
4. Repeat until one list becomes null
5. Attach remaining nodes

This is identical to the merge step in Merge Sort.


# Why We Use a Dummy Node

Without a dummy node:

- You must treat the first node separately
- Edge cases become messy

With dummy:

- No special case for head
- Cleaner code
- Fewer bugs

Dummy acts as a placeholder before the real head.


# Iterative Solution (Most Important)

Time Complexity: O(n + m)  
Space Complexity: O(1)

```java
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }

    if (l1 != null) {
        curr.next = l1;
    }

    if (l2 != null) {
        curr.next = l2;
    }

    return dummy.next;
}
```


# Visual Dry Run

l1: 1 → 3 → 5  
l2: 2 → 4 → 6  

Step 1:
Compare 1 and 2  
Attach 1  

Result:
dummy → 1  

Step 2:
Compare 3 and 2  
Attach 2  

Result:
dummy → 1 → 2  

Step 3:
Compare 3 and 4  
Attach 3  

Continue until one list ends.

Finally attach remaining part.


# Recursive Solution

Same logic, expressed recursively.

```java
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    if (l1 == null) return l2;
    if (l2 == null) return l1;

    if (l1.val <= l2.val) {
        l1.next = mergeTwoLists(l1.next, l2);
        return l1;
    } else {
        l2.next = mergeTwoLists(l1, l2.next);
        return l2;
    }
}
```

Time: O(n + m)  
Space: O(n + m) due to recursion stack


# 2️⃣ Merge K Sorted Lists

Problem:

Given k sorted linked lists, merge them into one sorted list.

Naive Approach:
Merge lists one by one  
Time: O(kN)

Better Approach:
Use Divide and Conquer

Steps:

1. Pair lists and merge
2. Reduce number of lists by half
3. Repeat until one list remains

Time Complexity: O(N log k)

Where:
N = total number of nodes  
k = number of lists  


# 3️⃣ Merge Sort on Linked List

Linked List can be sorted using Merge Sort.

Steps:

1. Find middle using Fast/Slow pointer
2. Split list into two halves
3. Recursively sort both halves
4. Merge sorted halves

Example:

4 → 2 → 1 → 3  

Split:
4 → 2  
1 → 3  

Sort each half:
2 → 4  
1 → 3  

Merge:
1 → 2 → 3 → 4  

Time: O(n log n)  
Space: O(log n) recursion stack


# Common Mistakes

❌ Forgetting to move curr pointer  
❌ Returning dummy instead of dummy.next  
❌ Not attaching remaining nodes  
❌ Losing reference to l1 or l2  
❌ Infinite loop due to wrong pointer movement  


# Edge Cases

✔ l1 is null  
✔ l2 is null  
✔ Both lists null  
✔ Duplicate values  
✔ Lists of different lengths  
✔ Single element lists  


# Recognizing Merge Pattern

Look for keywords:

- "Merge"
- "Sorted lists"
- "Combine lists"
- "K sorted lists"
- "Sort linked list"

If input lists are sorted → think merge.


# Final Mental Model

You are weaving two sorted chains together.

At each step:
Pick smaller node  
Attach it  
Move forward  

Repeat until done.

Merging is controlled pointer comparison.

Master merging and you unlock many advanced Linked List problems.
