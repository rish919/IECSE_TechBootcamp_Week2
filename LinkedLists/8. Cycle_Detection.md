# Cycle Detection in Linked List

Cycle detection is one of the most important Linked List problems.

It appears in:

- Detect if cycle exists
- Find start of cycle
- Linked List II problems
- Loop detection in systems

The standard solution uses Floyd’s Cycle Detection Algorithm,
also known as the Tortoise and Hare algorithm.


# 1️⃣ What Is a Cycle?

A cycle exists if a node’s next pointer eventually points back to a previous node.

Example:

1 → 2 → 3 → 4  
      ↑       ↓  
      ← ← ← ← ←  

Here, node 4 points back to node 2.

This creates an infinite loop.


# 2️⃣ Detecting a Cycle (Floyd’s Algorithm)

## Core Idea

Use two pointers:

- Slow pointer → moves 1 step
- Fast pointer → moves 2 steps

If there is no cycle:
Fast pointer reaches null.

If there is a cycle:
Fast pointer eventually meets slow pointer.


## Why They Meet

Inside a cycle:

- Slow moves 1 step
- Fast moves 2 steps

Fast gains 1 step per move.

Eventually, fast catches slow inside the loop.


# Code: Detect Cycle

```java
public boolean hasCycle(ListNode head) {
    if (head == null || head.next == null) {
        return false;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
```

Time Complexity: O(n)  
Space Complexity: O(1)


# 3️⃣ Visual Explanation

Example:

1 → 2 → 3 → 4  
      ↑       ↓  
      ← ← ← ← ←  

Step 1:
slow = 2  
fast = 3  

Step 2:
slow = 3  
fast = 2  

Step 3:
slow = 4  
fast = 4  

They meet → cycle detected.


# 4️⃣ Finding the Start of the Cycle

Once slow and fast meet,
we can find the cycle’s starting node.

## Key Insight

Let:

- L = distance from head to cycle start
- C = cycle length
- X = distance from cycle start to meeting point

When they meet:

2(L + X) = L + X + kC

Simplifying:

L = kC − X

This means:

If one pointer starts at head,
and one pointer starts at meeting point,
and both move 1 step at a time,

They will meet at the cycle start.


# Code: Find Cycle Start

```java
public ListNode detectCycle(ListNode head) {
    if (head == null || head.next == null) {
        return null;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            ListNode entry = head;

            while (entry != slow) {
                entry = entry.next;
                slow = slow.next;
            }

            return entry;
        }
    }

    return null;
}
```


# 5️⃣ Step-by-Step Finding Start

Example:

1 → 2 → 3 → 4 → 5  
          ↑       ↓  
          ← ← ← ← ←  

Suppose slow and fast meet at node 4.

Now:

Place one pointer at head (1)  
Keep other at meeting point (4)

Move both one step at a time.

They meet at node 3 → cycle start.


# 6️⃣ Edge Cases

✔ head == null  
✔ Single node without cycle  
✔ Single node with cycle  
✔ Two nodes with cycle  
✔ Large cycle  
✔ Entire list is cycle  


# 7️⃣ Common Mistakes

❌ Forgetting to check fast.next != null  
❌ Moving fast only one step  
❌ Not resetting pointer to head  
❌ Infinite loop due to incorrect condition  
❌ Comparing values instead of references  


Important:

Always compare nodes using:

```java
if (slow == fast)
```

NOT:

```java
if (slow.val == fast.val)
```


# 8️⃣ Why This Algorithm Is Powerful

- Uses constant space
- No HashSet required
- Elegant mathematical guarantee
- Works in O(n)

Alternative approach:
Use HashSet to store visited nodes.

But that uses O(n) extra space.


# 9️⃣ Recognizing Cycle Questions

Look for words like:

- Loop
- Cycle
- Infinite traversal
- Repeating nodes
- Detect repetition

Immediately think:

Fast and Slow pointers.


# Final Mental Model

Slow pointer walks normally.

Fast pointer runs ahead.

If there is no loop:
Fast hits null.

If there is a loop:
Fast eventually catches slow.

Then:

Reset one pointer to head.

Walk both one step at a time.

Where they meet → cycle start.

This is one of the most elegant Linked List algorithms.
