# Cycle Detection in Linked List

Cycle detection is one of the most important Linked List problems.

It appears in:

- Detect if a cycle exists
- Find the start of a cycle
- Loop detection problems
- Floyd’s Tortoise and Hare algorithm

We solve this using the Fast and Slow pointer technique.


# 1️⃣ What Is a Cycle?

A cycle exists if a node’s `next` pointer eventually points back to a previous node.

Example: Cycle starts at node 2

```
1 → 2 → 3 → 4
      ↑       ↓
      └───────┘
```

Here:
4 points back to 2.


# 2️⃣ Detecting a Cycle (Floyd’s Algorithm)

## Core Idea

Use two pointers:

- Slow pointer → moves 1 step
- Fast pointer → moves 2 steps

If there is no cycle:
Fast pointer reaches null.

If there is a cycle:
Fast pointer eventually meets slow pointer.


## Code: Detect Cycle

```java
public boolean hasCycle(ListNode head) {
    if (head == null || head.next == null) {
        return false;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
```

Time Complexity: O(n)  
Space Complexity: O(1)


# 3️⃣ Visual Pointer Movement

Example:

```
1 → 2 → 3 → 4
      ↑       ↓
      └───────┘
```

Initial:
slow = 1  
fast = 1  

Step 1:
slow = 2  
fast = 3  

Step 2:
slow = 3  
fast = 2  

Step 3:
slow = 4  
fast = 4  

They meet → cycle detected.


# 4️⃣ Why Do They Meet? (Mathematical Insight)

Let:

L = distance from head to cycle start  
C = length of cycle  
X = distance from cycle start to meeting point  

When slow and fast meet:

2(L + X) = L + X + kC  

Solving:

L = kC − X  

This means:

If one pointer starts from head,
and one pointer starts from the meeting point,
and both move 1 step at a time,

They will meet at the cycle start.


# 5️⃣ Finding the Start of the Cycle

After slow and fast meet:

1. Place one pointer at head
2. Keep the other at meeting point
3. Move both one step at a time
4. Where they meet is the cycle start


## Code: Find Cycle Start

```java
public ListNode detectCycle(ListNode head) {
    if (head == null || head.next == null) {
        return null;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            ListNode entry = head;

            while (entry != slow) {
                entry = entry.next;
                slow = slow.next;
            }

            return entry;
        }
    }

    return null;
}
```


# 6️⃣ Visual: Finding Cycle Start

Example:

```
1 → 2 → 3 → 4 → 5
          ↑       ↓
          └───────┘
```

Suppose slow and fast meet at node 4.

Now:

Pointer A → head (1)  
Pointer B → meeting point (4)

Move both 1 step at a time:

A = 2  
B = 5  

A = 3  
B = 3  

They meet at 3 → cycle start.


# 7️⃣ Edge Cases

✔ head == null  
✔ Single node without cycle  
✔ Single node with cycle  
✔ Two nodes cycle  
✔ Entire list is cycle  
✔ Long list with small loop  


# 8️⃣ Common Mistakes

❌ Forgetting to check `fast.next != null`  
❌ Moving fast only one step  
❌ Comparing values instead of node references  
❌ Not resetting pointer to head  
❌ Infinite loop due to incorrect condition  

Always compare nodes like this:

```java
if (slow == fast)
```

NOT:

```java
if (slow.val == fast.val)
```


# 9️⃣ Alternative Approach (Using HashSet)

You can store visited nodes in a HashSet.

If a node appears again → cycle exists.

Time: O(n)  
Space: O(n)

Floyd’s algorithm is better because it uses O(1) space.


# Final Mental Model

Slow walks.

Fast runs.

If there is no cycle:
Fast reaches null.

If there is a cycle:
Fast eventually catches slow.

Reset one pointer to head.

Move both one step.

They meet at cycle start.

This is one of the most elegant Linked List algorithms.
